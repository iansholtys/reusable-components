Table component

Demos of the Table component.

Minimal table

A simple table displaying basic object data with no searching or filtering.

Complex table

A table demonstrating more complex features:

  • Text search against searchable: true columns (SKU, category, status).
  • A custom field name so column headers can be customized separately from data property names.
  • Custom value, render and sort functions for the 'status' column to allow it to be driven by the 'quantity' column.
  • A custom filter UI element and function to allow filtering by status.
  • Action buttons to modify properties of each row to see re-rendering (including sorting and filtering).
  • A custom row click handler which logs data to the browser console.
  • defaultSort so the table opens sorted by computed status.

Mutable tables

With mutableData: false (default), each row is shallow-cloned into the table's internal array. With mutableData: true, object references remain intact, so changes can affect the original data.
